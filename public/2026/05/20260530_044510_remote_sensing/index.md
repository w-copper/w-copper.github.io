## 一、论文一：AgriFM - 面向农业制图的多源时序遥感基础模型

### 1.1 论文信息

| 项目 | 内容 |
|------|------|
| **标题** | AgriFM: A Multi-source Temporal Remote Sensing Foundation Model for Agriculture Mapping |
| **作者** | Wenyuan Li, Shunlin Liang, Keyan Chen, Yongzhe Chen, Han Ma, Jianglei Xu, Yichuan Ma, Shikang Guan, Husheng Fang, Zhenwei Shi |
| **机构** | 香港大学、北京航空航天大学等 |
| **发表时间** | 2025年5月 |
| **arXiv** | [2505.21357](https://arxiv.org/abs/2505.21357) |
| **代码** | [https://github.com/flyakon/AgriFM](https://github.com/flyakon/AgriFM) |

### 1.2 研究问题

准确的农作物制图（Crop Mapping）是精准农业和粮食安全的核心任务。然而，现有的遥感基础模型（RSFMs）在农业应用中面临两大挑战：

1. **多尺度时空建模不足**：农作物系统具有从田间纹理到景观级上下文的多尺度空间特征，以及从短期物候变化到整个生长季动态的多尺度时间特征。现有模型要么采用固定的时空窗口忽略多尺度特性，要么完全忽略时间信息仅关注空间模式。

2. **多源数据融合困难**：农业监测需要整合多种卫星数据源（如MODIS、Landsat-8/9、Sentinel-2），但这些数据具有不同的空间分辨率、时间分辨率和光谱特性，如何有效融合是一大难题。

### 1.3 解决方案

AgriFM提出了一种创新的多源时序遥感基础模型架构：

**核心创新：**

1. **改进的Video Swin Transformer架构**：
   - 将时间下采样与空间缩放操作同步进行
   - 实现层次化的时空特征提取
   - 支持长时间序列卫星数据的高效统一处理

2. **多源数据融合策略**：
   - 整合MODIS（高时间分辨率）、Landsat-8/9（中等分辨率）和Sentinel-2（高空间分辨率）三种数据源
   - 构建全球代表性数据集，包含超过2500万个图像样本
   - 利用土地覆盖产品进行监督预训练

3. **灵活的解码器架构**：
   - 动态融合学习到的时空表征
   - 支持多种下游任务（作物分类、产量预测等）

### 1.4 实验与结果

**数据集：**
- 预训练数据：全球2500万+图像样本
- 下游任务：多种农业制图基准数据集

**主要结果：**
- 在所有下游任务中均优于传统深度学习方法
- 超越现有的通用遥感基础模型（如Prithvi、SpectralGPT等）
- 验证了多源数据融合和时序建模对农业制图的重要性

### 1.5 评价与贡献

**学术贡献：**
- 首次针对农业制图任务设计专门的多源时序基础模型
- 提出了有效的时空特征同步提取策略
- 建立了大规模农业遥感预训练范式

**应用价值：**
- 可直接应用于精准农业中的作物监测
- 支持全球尺度的粮食产量预测
- 为农业保险和灾害评估提供技术支撑

## 二、论文二：Be the Change You Want to See - 重新审视遥感变化检测实践

### 2.1 论文信息

| 项目 | 内容 |
|------|------|
| **标题** | Be the Change You Want to See: Revisiting Remote Sensing Change Detection Practices |
| **作者** | Blaž Rolih, Matic Fučka, Filip Wolf, Luka Čehovin Zajc |
| **机构** | 卢布尔雅那大学（斯洛文尼亚） |
| **发表时间** | 2025年7月 |
| **arXiv** | [2507.03367](https://arxiv.org/abs/2507.03367) |
| **代码** | [https://github.com/blaz-r/BTC-change-detection](https://github.com/blaz-r/BTC-change-detection) |

### 2.2 研究问题

遥感变化检测（Change Detection）旨在定位同一地点不同时间拍摄图像之间的语义变化。近年来，研究者们不断为现有架构添加新的复杂组件以提升性能。然而，该论文指出了一个被忽视的关键问题：

**核心问题：** 大多数研究未能衡量**基础设计选择**（如骨干网络选择、预训练策略、训练配置）对性能的贡献，而这些基础选择往往比添加新的架构组件带来更显著的性能提升。

### 2.3 解决方案

该论文采取了一种"回归本质"的研究方法：

**研究方法：**

1. **系统性回顾设计空间**：
   - 深入分析变化检测模型的完整设计空间
   - 评估基础设计选择的性能贡献
   - 识别对新旧架构都有益的设计准则

2. **优化基线模型**：
   - 证明即使是架构简单的模型，经过精心设计也能达到或超越SOTA性能
   - 在6个具有挑战性的变化检测数据集上验证

3. **通用性设计准则**：
   - 提出的设计准则可推广到其他相关方法
   - 表明基础设计选择的空间尚未被充分探索

### 2.4 实验与结果

**评估数据集：**
- 6个具有挑战性的变化检测数据集
- 涵盖不同场景和变化类型

**主要发现：**
- 基础设计选择对性能的影响往往超过新架构组件
- 精心优化的简单模型可以匹配或超越复杂的SOTA方法
- 提出的最佳实践对其他相关方法同样有效

**关键结论：**
- 骨干网络选择、预训练策略、训练配置等基础选择至关重要
- 架构创新和核心组件优化同等重要

### 2.5 评价与贡献

**学术贡献：**
- 填补了变化检测领域基础设计选择研究的空白
- 提供了经过验证的最佳实践指南
- 为未来方法提供了坚实的研究基础

**实践价值：**
- 帮助研究者避免不必要的复杂性
- 提供即插即用的性能提升方案
- 促进变化检测研究的可复现性

## 四、2025年遥感AI发展趋势总结

基于本次论文调研，2025年遥感AI领域呈现以下发展趋势：

1. **基础模型持续演进**：从通用基础模型向领域专用模型发展（如AgriFM面向农业）

2. **多模态融合深化**：多源、多时相、多分辨率数据的融合成为标配

3. **回归本质的研究反思**：在追求架构创新的同时，重视基础设计选择的系统性研究

4. **开源生态繁荣**：顶级会议论文普遍提供完整代码和预训练模型

5. **实际应用导向**：研究更加注重实际部署和应用价值
