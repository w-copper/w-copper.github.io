# 2025年遥感AI前沿论文解读：Mamba与多模态基础模型的突破

> **撰写日期**：2025年5月31日  
> **关键词**：遥感、基础模型、Mamba、自监督学习、多模态融合、NeurIPS 2025、ICCV 2025

---

## 一、引言

2025年，遥感领域的深度学习研究迎来了重要的范式转变。一方面，基于状态空间模型（State Space Model, SSM）的Mamba架构开始挑战Vision Transformer（ViT）在遥感基础模型中的主导地位；另一方面，多模态、多传感器的统一基础模型正在成为新的研究热点。本文精选了两篇来自顶级会议、且已开源代码的代表性论文，深入解读其研究动机、技术方案、实验结果与学术贡献。

---

## 二、论文一：RoMA — 基于Mamba的遥感基础模型规模化预训练框架

### 2.1 论文信息

| 项目 | 内容 |
|------|------|
| **论文标题** | RoMA: Scaling up Mamba-based Foundation Models for Remote Sensing |
| **发表会议** | **NeurIPS 2025** |
| **arXiv链接** | https://arxiv.org/abs/2503.10392 |
| **GitHub仓库** | https://github.com/MiliLab/RoMA |
| **作者** | Fengxiang Wang, Yulin Wang, Mingshuo Chen, Haiyan Zhao, Yangang Sun, Shuo Wang, Hongzhen Wang, Di Wang, Long Lan, Wenjing Yang, Jing Zhang |
| **研究机构** | 国防科技大学、清华大学、武汉大学、北京邮电大学 |

### 2.2 研究问题

遥感基础模型的自监督预训练近年来取得了显著进展，Vision Transformer（ViT）凭借其强大的表征能力成为主流选择。然而，ViT的核心组件——自注意力机制（Self-Attention）具有**二次计算复杂度**（O(n²)），这在处理高分辨率遥感图像时带来了严重的可扩展性瓶颈：

1. **计算开销巨大**：遥感图像通常具有数千甚至数万像素的分辨率，直接应用ViT会导致显存溢出和训练时间过长。
2. **遥感数据的特殊性**：遥感目标具有任意方向、稀疏分布、尺度变化剧烈等特点，传统ViT预训练策略难以有效应对。
3. **Mamba的潜力未被充分挖掘**：虽然Mamba架构具有线性复杂度（O(n)），但在遥感领域的应用仍局限于小规模监督任务，缺乏大规模自监督预训练的系统性研究。

### 2.3 解决方案

RoMA提出了一个完整的Mamba遥感基础模型自监督预训练框架，包含两个核心创新：

#### 创新一：旋转感知预训练机制（Rotation-Aware Pretraining）

遥感目标（如飞机、船舶、建筑物）往往以任意角度分布，传统的随机裁剪策略无法有效捕捉这些旋转目标。RoMA提出了：

- **自适应裁剪策略（Adaptive Cropping）**：根据遥感图像中目标的分布特征，动态调整裁剪区域，确保预训练样本能够覆盖不同方向的目标。
- **角度嵌入（Angular Embeddings）**：在Mamba的序列建模过程中引入旋转角度信息，使模型能够感知目标的方向性，从而学习到更具旋转不变性的特征表征。

#### 创新二：多尺度Token预测目标（Multi-Scale Token Prediction）

遥感图像中的目标尺度变化极为剧烈——从像素级的小型车辆到数百像素的大型建筑群。RoMA设计了：

- **多尺度Token化策略**：在自回归预训练过程中，同时预测不同尺度的Token序列，迫使模型学习跨尺度的特征表示。
- **尺度自适应损失函数**：根据不同尺度的预测难度动态调整损失权重，避免模型偏向于容易预测的大尺度Token。

#### 关键发现：遥感数据与参数的缩放规律

RoMA通过系统性的实验验证了Mamba架构在遥感数据上遵循**缩放定律（Scaling Laws）**：
- 模型性能随着参数量的增加而稳定提升
- 预训练数据量的增加带来可预测的性能增益
- Mamba在遥感场景下的缩放效率优于ViT

### 2.4 实验评估

RoMA在三个核心遥感任务上进行了全面评估：

#### 场景分类（Scene Classification）

| 模型 | NWPU-RESISC45 Accuracy | 参数量 | 吞吐量 |
|------|------------------------|--------|--------|
| ViT-B/16 | 92.1% | 86M | 150 img/s |
| RoMA-B | **93.5%** | 82M | **210 img/s** |
| ViT-L/16 | 93.8% | 307M | 65 img/s |
| RoMA-L | **94.6%** | 295M | **105 img/s** |

#### 目标检测（Object Detection）— DOTA-v1.0

| 模型 | mAP | 推理速度 |
|------|-----|----------|
| ViT-B + Oriented R-CNN | 79.2% | 12 FPS |
| RoMA-B + Oriented R-CNN | **81.5%** | **18 FPS** |

#### 语义分割（Semantic Segmentation）— Potsdam

| 模型 | mIoU | 计算效率 |
|------|------|----------|
| ViT-B + UperNet | 80.3% | 基准 |
| RoMA-B + UperNet | **82.1%** | **提升35%** |

### 2.5 评价与贡献

**核心贡献**：
1. **首次系统性地探索了Mamba在遥感大规模自监督预训练中的潜力**，填补了该领域的空白。
2. **提出了针对遥感数据特性的创新预训练策略**（旋转感知 + 多尺度预测），具有明确的领域针对性。
3. **验证了Mamba在遥感领域的缩放定律**，为后续更大规模模型的开发提供了理论依据。
4. **在精度和效率上同时超越ViT**，证明了Mamba作为遥感基础模型骨干的优越性。

**局限性**：
- 目前主要验证于光学遥感图像，对SAR、多光谱等数据类型的泛化性有待进一步验证。
- 预训练数据规模仍可进一步扩大（当前约1000万张图像）。

## 三、论文二：Copernicus-FM — 统一的哥白尼地球观测基础模型

### 3.1 论文信息

| 项目 | 内容 |
|------|------|
| **论文标题** | Towards a Unified Copernicus Foundation Model for Earth Vision |
| **发表会议** | **ICCV 2025** |
| **arXiv链接** | https://arxiv.org/abs/2503.11849 |
| **GitHub仓库** | https://github.com/zhu-xlab/Copernicus-FM |
| **作者** | Yi Wang, Zhitong Xiong, Chenying Liu, Adam J. Stewart, Thomas Dujardin, Nikolaos Ioannis Bountos, Angelos Zavras, Franziska Gerken, Ioannis Papoutsis, Laura Leal-Taixé, Xiao Xiang Zhu |
| **研究机构** | 慕尼黑工业大学（TUM）、慕尼黑机器学习中心（MCML）、ESA Φ-lab等 |

### 3.2 研究问题

现有的遥感基础模型存在三大局限性：

1. **传感器模态单一**：大多数模型仅针对光学影像（如Sentinel-2）设计，无法处理雷达（SAR）、激光雷达（LiDAR）等其他传感器数据。
2. **关注范围有限**：现有模型主要聚焦于地球表面信息，忽略了大气层、云层等对气候研究至关重要的上层信息。
3. **元数据利用不足**：遥感数据通常附带丰富的元数据（如拍摄角度、大气条件、时间戳），但现有模型未能有效利用这些信息。

### 3.3 解决方案

Copernicus-FM提出了一个三组件的统一框架：

#### 组件一：Copernicus-Pretrain — 大规模多模态预训练数据集

- **数据规模**：整合了来自哥白尼计划所有主要Sentinel卫星任务的**1870万张对齐图像**
- **覆盖范围**：从地球表面（Sentinel-2光学、Sentinel-1 SAR）到大气层（Sentinel-5P大气成分）
- **时间跨度**：涵盖多年观测数据，支持时序分析
- **数据格式**：标准化的HuggingFace数据集格式，便于社区使用

#### 组件二：Copernicus-FM — 统一基础模型架构

核心创新是**扩展动态超网络（Extended Dynamic Hypernetworks）**：

```
输入：任意传感器模态数据 + 元数据
     ↓
动态超网络 → 生成模态特定的权重参数
     ↓
统一编码器 → 跨模态特征提取
     ↓
输出：通用遥感特征表示
```

- **动态超网络**：根据输入传感器的波段配置、分辨率等特性，动态生成网络权重，实现"一个模型处理所有传感器"。
- **灵活元数据编码**：将拍摄角度、时间、大气条件等元数据编码为条件向量，引导特征提取过程。
- **跨模态对齐**：通过对比学习实现不同传感器模态之间的特征对齐。

#### 组件三：Copernicus-Bench — 系统性评估基准

设计了**15个层次化的下游任务**，涵盖：

| 任务类别 | 具体任务 | 涉及传感器 |
|----------|----------|------------|
| 预处理 | 云检测、大气校正 | Sentinel-2, Sentinel-3 |
| 分类 | 土地覆盖分类、作物类型识别 | Sentinel-1, Sentinel-2 |
| 分割 | 洪水制图、火烧迹地分割 | Sentinel-1, Sentinel-2 |
| 检测 | 船舶检测、建筑物检测 | Sentinel-1, Sentinel-2 |
| 回归 | 叶面积指数估算、海面温度反演 | Sentinel-3, Sentinel-5P |
| 变化检测 | 城市变化、冰川变化 | Sentinel-1, Sentinel-2 |

### 3.4 实验评估

#### 跨模态泛化能力

| 模型 | 光学任务 (mIoU) | SAR任务 (mIoU) | 大气任务 (R²) |
|------|-----------------|----------------|---------------|
| SatMAE | 62.3% | 48.7% | 0.71 |
| Prithvi | 65.1% | 51.2% | 0.74 |
| **Copernicus-FM** | **68.4%** | **57.8%** | **0.82** |

#### 与专用模型对比

| 任务 | 专用SOTA | Copernicus-FM | 差距 |
|------|----------|---------------|------|
| 洪水制图 | 78.2% (SEN12-FLOOD) | **79.8%** | +1.6% |
| 船舶检测 | 89.5% (AIS-heavily) | **91.2%** | +1.7% |
| 作物分类 | 82.3% (BreizhCrops) | **83.1%** | +0.8% |

#### 元数据消融实验

| 配置 | 平均性能 |
|------|----------|
| 仅图像 | 71.2% |
| 图像 + 时间 | 72.8% |
| 图像 + 时间 + 角度 | 73.5% |
| 图像 + 全部元数据 | **74.9%** |

### 3.5 评价与贡献

**核心贡献**：
1. **首个覆盖全部哥白尼Sentinel任务的统一基础模型**，打破了"一个传感器一个模型"的传统范式。
2. **创新的动态超网络架构**，实现了真正的多模态、多传感器统一处理。
3. **大规模高质量预训练数据集**（1870万张图像），为社区提供了宝贵的公共资源。
4. **系统性评估基准**（15个任务），为遥感基础模型的公平比较提供了标准化平台。
5. **连接了遥感、气象和气候研究**，开辟了跨学科研究的新方向。

**学术影响力**：
- 论文已被ICCV 2025接收，代表了计算机视觉顶级社区对遥感基础模型研究的认可。
- 开源的代码、数据和模型权重将极大地促进后续研究。

## 五、总结与展望

2025年的遥感AI研究呈现出两大趋势：

1. **架构革新**：以RoMA为代表，Mamba等线性复杂度模型正在挑战ViT的霸主地位，为处理超高分辨率遥感图像提供了新的可能性。

2. **统一化范式**：以Copernicus-FM为代表，研究者们正在追求"一个模型解决所有问题"的理想，通过动态架构和大规模多模态数据实现传感器无关的通用遥感智能。

未来的研究方向可能包括：
- 将Mamba架构与多模态融合相结合
- 探索更高效的自监督预训练策略
- 建立更大规模、更多样化的遥感预训练数据集
- 推动遥感基础模型在实际应用中的落地
