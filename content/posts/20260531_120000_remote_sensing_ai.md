### 2. 研究问题

遥感基础模型（RSFM）近年来取得显著进展，但面临以下核心挑战：

1. **计算复杂度瓶颈**：主流的Vision Transformer（ViT）架构具有二次复杂度（O(N²)），在处理高分辨率遥感图像（如DOTA数据集中的4000×4000像素图像）时计算开销巨大。

2. **Mamba架构的预训练空白**：Mamba架构以其线性复杂度（O(N)）展现出巨大潜力，但在遥感领域的应用仅限于小规模监督学习任务，缺乏大规模自监督预训练框架。

3. **遥感图像的独特挑战**：
   - 前景目标稀疏分布，背景复杂
   - 目标具有任意方向（不同于自然图像受重力约束）
   - 目标尺度变化极端

### 3. 解决方案：RoMA框架

RoMA（Rotation-aware Multi-scale Autoregressive learning）提出了一套针对Mamba架构的自监督预训练框架，包含两大核心创新：

#### 3.1 旋转感知预训练机制（Adaptive Rotation Encoding Strategy）

- **自适应区域裁剪**：利用LBP特征描述符识别高价值区域，而非随机裁剪，确保旋转操作不会破坏目标信息
- **角度嵌入（Angular Embedding）**：在旋转裁剪中引入可学习的角度嵌入，帮助模型感知角度变化
- **旋转预测任务**：要求模型在自回归预训练中预测角度变化，强化旋转不变性特征学习

#### 3.2 多尺度Token预测策略（Multi-scale Token Prediction）

- **问题**：遥感图像的扁平化处理会破坏空间连续性，导致表面信息表达不完整
- **方案**：在不同空间尺度上聚合预测token信息，帮助Mamba捕获更完整、结构更鲁棒的目标表示
- **损失函数**：结合标准token级MSE损失与高尺度聚合块MSE损失，通过λ参数平衡两者贡献

#### 3.3 自回归预训练范式

- **与MAE的对比**：MAE的掩码操作会破坏Mamba的线性扫描机制，而自回归预训练天然适配Mamba的序列处理特性
- **KV缓存预测**：Mamba编码器处理整个图像计算Key和Value，通过可学习的Query向量进行下一token预测

### 4. 实验结果

#### 4.1 缩放定律验证

RoMA首次验证了Mamba架构在遥感领域遵循数据和参数缩放定律：

- **数据缩放**：从62.5K到4M样本，性能持续提升，无明显瓶颈
- **参数缩放**：从Tiny到Large模型，下游任务性能稳定增长

### 5. 评估与贡献

**核心贡献**：
1. 首个面向遥感的Mamba架构自监督预训练框架
2. 旋转感知机制有效解决遥感目标方向多样性问题
3. 多尺度预测策略应对极端尺度变化

## 论文二：RemoteSAM — 面向地球观测的统一分割基础模型

### 1. 论文信息

| 项目 | 内容 |
|------|------|
| **标题** | RemoteSAM: Towards Segment Anything for Earth Observation |
| **作者** | Liang Yao, Fan Liu, Delong Chen, Chuanyi Zhang 等 |
| **单位** | 河海大学、香港科技大学、东南大学 |
| **会议** | ACM Multimedia 2025（口头报告） |
| **arXiv** | [2505.18022](https://arxiv.org/abs/2505.18022) |
| **代码** | [https://github.com/1e12Leon/RemoteSAM](https://github.com/1e12Leon/RemoteSAM) |

### 2. 研究问题

遥感视觉基础模型面临三大挑战：

1. **任务碎片化**：现有方法为每个任务（分类、检测、分割等）设计专用模型和头部，知识共享有限

2. **文本统一范式的局限**：基于VLM的方法（如Falcon、GeoChat）在像素级预测任务上表现不佳，因为自然语言不适合表示密集像素输出

3. **数据语义覆盖不足**：现有遥感指代表达分割数据集规模小、类别有限、属性描述单一

### 3. 解决方案：RemoteSAM

#### 3.1 核心设计思想：以指代表达分割（RES）为中心

- **像素级mask作为原子单元**：所有任务都可以从像素级预测向上兼容
- **消除LLM骨干**：相比VLM方法，参数量从数十亿降至百万级
- **统一架构**：单一模型支持分割、检测、分类、定位、计数等多种任务

#### 3.2 自动数据引擎：RemoteSAM-270K

- **规模**：270K图像-文本-mask三元组，297个类别，16种属性类型
- **构建流程**：
  1. **数据集整合**：集成iSAID、LoveDA、DOTA、HRRSD等多个遥感数据集
  2. **VLM生成表达**：利用Qwen2-VL-72B生成丰富语义描述
  3. **混合教师框架**：结合GroundedSAM2、RMSIN等专家模型生成伪标签
  4. **质量过滤**：利用SigLIP2计算图文相似度，迭代去除低质量样本

- **三种生成策略**：
  - One-to-One：直接整合现有指代分割标注
  - One-to-Many：为每个类别生成独立mask
  - One-to-Zero：创建空mask配对不存在类别的描述

#### 3.3 任务统一范式

RemoteSAM通过任务转换函数将RES输出转换为不同任务格式：

| 任务级别 | 任务类型 | 转换方法 |
|----------|----------|----------|
| 像素级 | 指代分割 | 直接输出mask |
| 像素级 | 语义分割 | 迭代聚合所有类别mask |
| 区域级 | 视觉定位 | Mask-to-BBox转换 |
| 区域级 | 目标检测 | EPOC轮廓检测 + M2B |
| 图像级 | 多标签分类 | 空间加权概率池化 |
| 图像级 | 图像描述 | 规则化生成 |

## 总结与展望

### 两篇论文的共同趋势

1. **效率优先**：RoMA通过Mamba架构实现线性复杂度，RemoteSAM通过消除LLM骨干大幅降低参数量
2. **遥感特化**：两者都针对遥感图像的独特特性（任意方向、多尺度、稀疏目标）进行专门设计
3. **开源贡献**：均提供完整代码和预训练模型，促进社区发展

### 未来方向

1. **多模态融合**：将Mamba架构与多模态遥感数据（光学+SAR+高光谱）结合
2. **时序建模**：扩展到时序遥感数据，支持变化检测和动态监测
3. **边缘部署**：进一步优化模型效率，支持星上实时处理
4. **数据引擎升级**：利用更强大的VLM持续扩展数据集规模和语义覆盖
